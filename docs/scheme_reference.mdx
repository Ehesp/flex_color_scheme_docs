# Scheme Reference

This table lists all current built-in color schemes in FlexColorScheme, the version it was released with, its enum value, short name and description. It shows a small thumbnail of each color scheme in light and dark theme mode. Clicking on a thumbnail opens a high-resolution version of the image.

The column **Swap on M3** shows if a scheme will when the `FlexScheme.light/dark` and `FlexThemeData.light/dark` properties `swapLegacyOnMaterial3` and `useMaterial3` are set to true, swap its `secondary`, `tertiary` colors and their container colors. This is only set to true, if doing so makes a scheme originally designed for the Material 2 color system, a better fit for intended color design when using the Material 3 color system. If it does not improve the fit or if it is already a good fit, it is not set to swap. The indicated swap only happens if you also set both `swapLegacyOnMaterial3` and `useMaterial3` to `true`.

Color scheme designed released starting from version 5.0 were designed with fit and usage in the Material 3 color system in mind. They can also be native Material 3 schemes based on seeded colors or slight variations of them. It also includes color schemes used as examples in the Material 3 design guide, most prominently the one often referred to as M3 baseline.

FlexColorScheme has the capability to use the secondary color of a built-in scheme as a key color to seed generate also the secondary colors. The secondary color and its container, are in the Material 3 color system used for less prominent components in the UI. It is in the default expected design a less chromatic version of the primary color's hue, or a hue close to it. If the hue of a scheme's secondary color, or your custom colors, differs a lot in hue from the primary color, then it may be preferred to not use the secondary color as key color when seed generating a color scheme for a better M3 design fit. The Flutter SDK locked down seed generation offered by `ColorScheme.fromSeed` can always only use one seed color. The tertiary colors don't have such restrictions so there using it as key color or relaying on the default Material 3 color systems +60 degree hue rotation, is a matter of design choice, either will work fine, just gives different results.

The column **No secondary key** indicate if it is preferred to exclude using the scheme's secondary color when making a seed generated Material 3 color system color scheme from it, to get a scheme that fits the secondary color's design intent in the Material 3 color system design.
